### 一、哈希表解法的核心思路
暴力法的低效在于“重复检查”——对每个数都要遍历后面的所有数找匹配值；而哈希表的核心是**用空间换时间**，通过记录“已经遍历过的数及其下标”，把“找匹配值”的操作从 $O(n)$ 降到 $O(1)$，具体思路如下：
1. **问题转化**：对于当前遍历的数 `nums[i]`，我们需要找的是 `target - nums[i]`（这个值称为“补数”），只要这个补数在之前遍历过的数中存在，就能凑出 `target`。
2. **哈希表的作用**：存储**已经遍历过的数**和它的**下标**（键：数值，值：数值对应的下标），相当于一个“快速查询字典”。
3. **遍历逻辑**：
   - 遍历每个数时，先查哈希表中是否有当前数的“补数”；
   - 如果有，直接返回补数的下标和当前数的下标；
   - 如果没有，把当前数和它的下标存入哈希表，继续遍历下一个数。

简单来说：暴力法是“找别人”，哈希表法是“让别人找自己”——遍历到第 `i` 个数时，只关心前面有没有数能和自己凑成 `target`，而哈希表帮你快速验证这一点。

### 二、代码逐行解析
```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        // 1. 定义哈希表：键是数组中的数值，值是数值对应的下标
        unordered_map<int, int> hashtable;
        // 2. 遍历数组，i是当前数的下标
        for (int i = 0; i < nums.size(); ++i) {
            // 3. 计算当前数的补数：target - nums[i]
            auto it = hashtable.find(target - nums[i]);
            // 4. 检查补数是否存在于哈希表中（即是否在之前遍历过的数里）
            if (it != hashtable.end()) {
                // 存在则返回：补数的下标（it->second） + 当前数的下标（i）
                return {it->second, i};
            }
            // 5. 补数不存在，把当前数和下标存入哈希表，供后续数查询
            hashtable[nums[i]] = i;
        }
        // 6. 题目保证有解，此返回仅为语法完整性
        return {};
    }
};
```

### 三、举例模拟执行过程
还是用 `nums = [2,7,11,15]`，`target = 9` 举例，一步步看哈希表的变化：
1. **i=0，nums[i]=2**：
   - 补数 = 9 - 2 = 7；
   - 哈希表为空，找不到7；
   - 把 `{2:0}` 存入哈希表。
2. **i=1，nums[i]=7**：
   - 补数 = 9 - 7 = 2；
   - 查哈希表，找到2（对应下标0）；
   - 直接返回 `[0,1]`，遍历结束。

### 四、哈希表解法的优势
| 维度         | 暴力法       | 哈希表法     |
|--------------|--------------|--------------|
| 时间复杂度   | $O(n^2)$     | $O(n)$（每个数仅遍历1次，哈希表查询/插入都是$O(1)$） |
| 空间复杂度   | $O(1)$（无额外空间） | $O(n)$（最坏情况存储所有数） |
| 实际效率     | 数据量大时极慢 | 高效，工程首选 |

### 关键细节说明
- 为什么用 `unordered_map` 而不是 `map`？
  `unordered_map` 是哈希表实现，查询/插入的平均时间复杂度是 $O(1)$；而 `map` 是红黑树实现，时间复杂度是 $O(logn)$，效率更低。
- 为什么先查补数再存当前数？
  避免“同一个数被重复使用”（比如 `target=4`，`nums=[2]`，如果先存再查，会错误匹配自己）。

### 总结
1. 哈希表解法的核心是**补数思想 + 快速查询**：把“找两个数之和等于target”转化为“找当前数的补数是否已出现”，用哈希表实现补数的快速查询。
2. 执行逻辑：遍历中先查补数（查历史）→ 找到则返回 → 没找到则存当前数（留未来）。
3. 核心优势：用 $O(n)$ 的空间代价，把时间复杂度从 $O(n^2)$ 优化到 $O(n)$，是两数之和的最优解法。
