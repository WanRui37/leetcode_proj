### 一、问题背景与核心思路
**两数之和问题**：给定一个整数数组 `nums` 和一个目标值 `target`，要求找出数组中**两个不同位置**的数，使其和等于 `target`，并返回这两个数的下标。

这段代码采用的是**暴力枚举法（双重循环）**，核心思路非常直观：
1. 遍历数组中的每一个元素 `nums[i]`；
2. 对于每一个 `nums[i]`，再遍历它后面的所有元素 `nums[j]`（`j > i`，避免重复检查同一对元素）；
3. 检查 `nums[i] + nums[j]` 是否等于目标值 `target`，若满足则直接返回下标 `[i, j]`；
4. 若遍历完所有组合都未找到，则返回空数组（题目保证有唯一解，此情况仅为鲁棒性设计）。

### 二、代码逐行解析
```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        // 1. 获取数组长度，避免重复调用nums.size()，提升效率
        int n = nums.size();
        // 2. 外层循环：遍历数组中的每一个元素，下标为i
        for (int i = 0; i < n; ++i) {
            // 3. 内层循环：遍历i之后的所有元素，下标为j（j从i+1开始，避免i和j重复）
            for (int j = i + 1; j < n; ++j) {
                // 4. 核心判断：当前两个数的和是否等于目标值
                if (nums[i] + nums[j] == target) {
                    // 5. 找到解，直接返回两个下标（按i在前、j在后的顺序）
                    return {i, j};
                }
            }
        }
        // 6. 题目保证有解，此返回仅为语法完整性
        return {};
    }
};
```

### 三、举例理解执行过程
假设输入：`nums = [2,7,11,15]`，`target = 9`
1. 外层循环 `i=0`（对应数值 2）：
   - 内层循环 `j=1`（对应数值 7）：检查 `2+7=9`，等于目标值，直接返回 `[0,1]`。
2. 后续循环（`i=1/2/3`）不会执行，因为已经找到解并返回。

### 四、方法优缺点
| 优点                | 缺点                  |
|---------------------|-----------------------|
| 逻辑简单，容易理解  | 时间复杂度高（$O(n^2)$） |
| 无需额外空间（$O(1)$） | 数据量大时效率极低    |

### 总结
1. 这段代码的核心是**双重循环暴力枚举**：外层固定一个数，内层检查其后的所有数，直到找到和为目标值的两个数。
2. 关键细节：`j` 从 `i+1` 开始，避免重复检查（如 `i=1,j=0` 与 `i=0,j=1` 是同一对）。
3. 暴力法适合理解问题本质，但实际工程中更推荐用哈希表（$O(n)$ 时间复杂度）优化，这也是两数之和的最优解思路。
