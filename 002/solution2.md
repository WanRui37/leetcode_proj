### 一、问题背景与核心思路
**字母异位词分组问题**：给定一个字符串数组 `strs`，将所有**字母异位词**分到同一组（字母异位词指字母相同但排列不同的字符串，比如 "eat"、"tea"、"ate"）。

这段代码的核心思路是：
1. **特征提取**：为每个字符串生成一个**唯一的特征标识**——用长度为26的数组统计每个小写字母出现的次数（比如 "eat" 的特征是 `a:1, e:1, t:1`，其余字母为0），字母异位词的特征标识完全相同。
2. **哈希分组**：用哈希表存储“特征标识 → 对应异位词列表”的映射，遍历所有字符串时，根据特征标识将其归类到对应的组中。
3. **结果整理**：将哈希表中的所有异位词列表提取出来，组成最终结果。

简单来说：**把“字母排列不同但组成相同”的字符串，通过“字符计数”转化为“相同的特征键”，再用哈希表按键分组**。

### 二、代码逐行解析
```cpp
class Solution {
public:
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        // 1. 自定义哈希函数：因为C++的unordered_map不支持直接用array<int,26>作为键，需要自定义哈希
        auto arrayHash = [fn = hash<int>{}] (const array<int, 26>& arr) -> size_t {
            // accumulate：累加数组中所有元素，生成一个唯一的哈希值
            // acc是累加结果，num是当前数组元素；(acc << 1) ^ fn(num) 保证不同数组的哈希值不同
            return accumulate(arr.begin(), arr.end(), 0u, [&](size_t acc, int num) {
                return (acc << 1) ^ fn(num);
            });
        };

        // 2. 定义哈希表：
        // 键：array<int,26>（字符计数数组），值：vector<string>（对应异位词列表）
        // decltype(arrayHash)：推导自定义哈希函数的类型；0是初始桶数，arrayHash是自定义哈希函数
        unordered_map<array<int, 26>, vector<string>, decltype(arrayHash)> mp(0, arrayHash);
        
        // 3. 遍历所有字符串，逐个分组
        for (string& str: strs) {
            // 初始化字符计数数组，26个元素默认值为0（对应a-z）
            array<int, 26> counts{};
            int length = str.length();
            // 统计当前字符串中每个字符的出现次数
            for (int i = 0; i < length; ++i) {
                // str[i] - 'a'：将字符（如'a'）转化为0-25的索引（如a→0，b→1...z→25）
                counts[str[i] - 'a'] ++;
            }
            // 将当前字符串加入对应特征键的列表中
            mp[counts].emplace_back(str);
        }

        // 4. 整理哈希表中的结果，转化为最终的二维数组
        vector<vector<string>> ans;
        for (auto it = mp.begin(); it != mp.end(); ++it) {
            // 将每个异位词组加入结果数组
            ans.emplace_back(it->second);
        }
        return ans;
    }
};
```

### 三、关键细节拆解
#### 1. 为什么用“字符计数数组”作为特征键？
- 字母异位词的核心特征是“每个字母出现的次数完全相同”，比如：
  - "eat" → counts = [1,0,0,0,1,0,...0,1,0,...]（a:1, e:1, t:1）
  - "tea" → counts = [1,0,0,0,1,0,...0,1,0,...]（和eat完全一样）
- 相比“排序字符串”（比如将"eat"排序为"aet"作为键），字符计数的时间效率更高（排序的时间是 $O(klogk)$，k是字符串长度；计数是 $O(k)$）。

#### 2. 为什么需要自定义哈希函数？
- C++ 的 `unordered_map` 默认不支持 `array<int, 26>` 作为键——因为标准库没有为 `array<int, 26>` 提供哈希函数。
- 自定义的 `arrayHash` 函数通过**累加+异或**的方式，将26个整数的数组转化为一个唯一的 `size_t` 类型哈希值，确保相同的计数数组对应相同的哈希值，不同的计数数组对应不同的哈希值。

### 四、举例模拟执行过程
假设输入 `strs = ["eat", "tea", "tan", "ate", "nat", "bat"]`，执行过程如下：
1. 处理 "eat"：
   - 计数数组：a=1, e=1, t=1，其余0 → 哈希表中新增键（该计数数组），值为 ["eat"]。
2. 处理 "tea"：
   - 计数数组和"eat"完全相同 → 哈希表中对应键的值变为 ["eat", "tea"]。
3. 处理 "tan"：
   - 计数数组：a=1, n=1, t=1 → 哈希表新增键，值为 ["tan"]。
4. 处理 "ate"：
   - 计数数组和"eat"相同 → 对应值变为 ["eat", "tea", "ate"]。
5. 处理 "nat"：
   - 计数数组和"tan"相同 → 对应值变为 ["tan", "nat"]。
6. 处理 "bat"：
   - 计数数组：a=1, b=1, t=1 → 哈希表新增键，值为 ["bat"]。
7. 整理结果：
   - 最终 ans = [["eat","tea","ate"], ["tan","nat"], ["bat"]]。

### 五、复杂度分析
| 维度         | 时间复杂度                          | 空间复杂度                          |
|--------------|-------------------------------------|-------------------------------------|
| 分析         | $O(nk)$（n是字符串数量，k是字符串最大长度；每个字符串计数耗时$O(k)$，哈希表操作$O(1)$） | $O(nk)$（哈希表存储所有字符串，计数数组占$O(26)=O(1)$，可忽略） |

### 总结
1. 核心思路：**字符计数定特征 + 哈希表分组**——用26位字符计数数组作为字母异位词的唯一标识，通过哈希表将同特征的字符串归为一组。
2. 关键技巧：自定义哈希函数解决 `array<int,26>` 作为 `unordered_map` 键的问题，相比排序法更高效。
3. 执行逻辑：遍历字符串→统计字符计数→按计数分组→整理分组结果。
