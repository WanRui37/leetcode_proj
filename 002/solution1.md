### 一、问题背景与核心思路
**字母异位词**：指由相同字母组成，但排列顺序不同的字符串（比如 "eat"、"tea"、"ate"）。
**问题要求**：将输入数组中所有字母异位词分到同一组，最终返回分组后的二维数组。

这段代码的核心思路是**“特征提取 + 哈希表分组”**：
1. **特征提取**：对于任意一个字母异位词，把它的字符按字典序排序后，会得到一个**唯一的、相同的字符串**（比如 "eat" 和 "tea" 排序后都是 "aet"），这个排序后的字符串就是该组异位词的“特征键（key）”。
2. **哈希表分组**：用哈希表存储“特征键 → 对应异位词列表”的映射关系，遍历所有字符串时：
   - 对当前字符串排序得到特征键；
   - 将当前字符串加入该特征键对应的列表中；
3. 最后把哈希表中所有的列表取出，就是最终的分组结果。

简单来说：**把“字母异位词”转化为“相同的特征键”，再用哈希表按键分组**。

### 二、代码逐行解析
```cpp
class Solution {
public:
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        // 1. 定义哈希表：键是排序后的特征字符串，值是该特征对应的所有异位词
        unordered_map<string, vector<string>> mp;
        
        // 2. 遍历输入的每一个字符串
        for (string& str: strs) {
            // 3. 复制当前字符串，用于生成特征键（避免修改原字符串）
            string key = str;
            // 4. 对key排序，得到当前字符串的特征键（异位词排序后key相同）
            sort(key.begin(), key.end());
            // 5. 将当前字符串加入对应key的列表中（emplace_back比push_back更高效）
            mp[key].emplace_back(str);
        }
        
        // 6. 定义结果数组，存储最终的分组
        vector<vector<string>> ans;
        // 7. 遍历哈希表，把每个key对应的列表加入结果数组
        for (auto it = mp.begin(); it != mp.end(); ++it) {
            ans.emplace_back(it->second);
        }
        
        // 8. 返回最终分组结果
        return ans;
    }
};
```

### 三、举例模拟执行过程
假设输入：`strs = ["eat", "tea", "tan", "ate", "nat", "bat"]`，一步步看执行过程：
1. 遍历 "eat"：
   - key = "eat" → 排序后为 "aet"；
   - mp["aet"] = ["eat"]。
2. 遍历 "tea"：
   - key = "tea" → 排序后为 "aet"；
   - mp["aet"] = ["eat", "tea"]。
3. 遍历 "tan"：
   - key = "tan" → 排序后为 "ant"；
   - mp["ant"] = ["tan"]。
4. 遍历 "ate"：
   - key = "ate" → 排序后为 "aet"；
   - mp["aet"] = ["eat", "tea", "ate"]。
5. 遍历 "nat"：
   - key = "nat" → 排序后为 "ant"；
   - mp["ant"] = ["tan", "nat"]。
6. 遍历 "bat"：
   - key = "bat" → 排序后为 "abt"；
   - mp["abt"] = ["bat"]。
7. 最终哈希表中的内容：
   - "aet": ["eat", "tea", "ate"]
   - "ant": ["tan", "nat"]
   - "abt": ["bat"]
8. 把这些列表存入ans，返回：`[["eat","tea","ate"],["tan","nat"],["bat"]]`。

### 四、复杂度分析
| 维度         | 分析                                                                 |
|--------------|----------------------------------------------------------------------|
| 时间复杂度   | $O(n * k logk)$：n是字符串数量，k是单个字符串的最大长度；排序每个字符串的时间是$O(k logk)$，遍历n个字符串总时间为$O(n*k logk)$。 |
| 空间复杂度   | $O(n * k)$：哈希表需要存储所有字符串，最坏情况下无重复异位词，空间为所有字符串的长度之和。 |

### 五、关键细节补充
1. 为什么用 `unordered_map` 而非 `map`？
   `unordered_map` 的插入/查询平均时间复杂度是 $O(1)$，而 `map` 是 $O(logn)$，效率更高，适合分组场景。
2. 有没有更优的特征提取方式？
   可以用“字符计数”作为特征键（比如 "eat" 对应 "a1e1t1"），时间复杂度可降到 $O(n*k)$（无需排序），但实现稍复杂，新手先掌握排序法即可。
3. `emplace_back` 和 `push_back` 的区别？
   `emplace_back` 直接在容器内构造对象，减少拷贝，效率更高；`push_back` 是先构造对象再拷贝/移动，这里用 `emplace_back` 更优。

### 总结
1. 字母异位词分组的核心是**提取唯一特征键**：排序后的字符串是异位词的“统一标识”，能将不同排列的异位词映射到同一组。
2. 核心流程：遍历字符串 → 生成特征键 → 哈希表分组 → 提取分组结果。
3. 排序法实现简单、易理解，是解决该问题的基础方法，时间复杂度主要由字符串排序决定。
